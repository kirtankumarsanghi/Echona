# Contributing to ECHONA

Thank you for your interest in contributing to ECHONA! We welcome contributions from the community.

## üöÄ Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/echona-pro.git
   cd echona-pro
   ```
3. **Create a branch** for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## üíª Development Setup

Follow the installation instructions in [README.md](./README.md#installation) to set up your development environment.

## üìù Making Changes

### Code Style
- **Frontend (React)**: Follow React best practices, use functional components and hooks
- **Backend (Node.js)**: Use ES6+ features, async/await for asynchronous code
- **Python (ML)**: Follow PEP 8 style guidelines
- **Formatting**: Use consistent indentation (2 spaces for JS/JSX, 4 spaces for Python)

### Commit Messages
Use clear, descriptive commit messages:
```
feat: Add voice emotion detection feature
fix: Resolve Spotify authentication bug
docs: Update installation instructions
style: Format dashboard component
refactor: Simplify mood detection logic
```

### Testing
- Test your changes thoroughly before submitting
- Ensure all existing functionality still works
- Add tests for new features when applicable

## üîç Pull Request Process

1. **Update documentation** if you're adding new features
2. **Update the README.md** with details of changes if needed
3. **Ensure your PR description** clearly describes the problem and solution
4. **Link any relevant issues** in your PR description
5. **Request review** from maintainers

### PR Checklist
- [ ] Code follows the project's style guidelines
- [ ] Self-review of the code completed
- [ ] Comments added for complex logic
- [ ] Documentation updated
- [ ] No new warnings or errors introduced
- [ ] Tested on your local environment

## üêõ Reporting Bugs

### Before Submitting
- Check if the bug has already been reported in Issues
- Collect information about the bug (browser, OS, steps to reproduce)

### Bug Report Template
```markdown
**Describe the bug**
A clear description of what the bug is.

**To Reproduce**
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '...'
3. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots.

**Environment:**
 - OS: [e.g., Windows 11]
 - Browser: [e.g., Chrome 120]
 - Node version: [e.g., 18.0.0]
 - Python version: [e.g., 3.10]
```
